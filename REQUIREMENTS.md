# 顧客中心（Salesforce型）CRM 要件定義 v1（確定）

## 0. 決定事項
- 課金の粒度：**1案件に明細行を複数**（初期費用=単発 ＋ 月額=定額 の混在を明細で表現）
- リカーリング：**契約期間・解約(churn)まで管理**
- 連絡先：**単一のまま**
- 締結を境に **商談（締結前）↔ 顧客（締結後）** を分ける
- 単発売上：**受注済みベース**で集計

## 1. エンティティ構造
```
顧客(Account) 1──N 商談/契約(Deal) 1──N 明細(DealLineItem)
                          └─ 1──N 活動ログ(Activity)
```
- **商談(締結前)** … 受注を目指すパイプライン。
- **契約(締結後)** … 受注した Deal ＝ 顧客の契約。顧客配下に明細（MRR/単発）がぶら下がる。
- 同じ Deal がフェーズ進行で「商談 → 契約」へ変わる（別テーブルにしない）。

## 2. フェーズ（締結が境界）
- **締結前（商談・カンバン対象）**：`初回接触` → `提案` → `条件調整` → `契約`(=受注/締結)
- **締結後（顧客側で管理）**：`契約` → `オンボーディング` → `運用中` →（`解約`は明細のchurnで表現）
- **商談クローズ**：`保留` / `失注`
- カンバン列＝ 初回接触・提案・条件調整・契約(受注)・保留・失注。
  `オンボーディング`/`運用中` はカンバンに出さず、**顧客詳細**で管理。
- 判定：`契約後(顧客) = フェーズ ∈ {契約, オンボーディング, 運用中}`

## 3. 明細(DealLineItem)
| 項目 | 内容 |
|---|---|
| サービス名/品目 | 例: TTS運用、動画制作、初期構築 |
| 課金タイプ | `月次定額` / `単発` |
| 金額(単価) | 月次定額→月額、単発→総額 |
| 数量 | 既定1（小計=単価×数量） |
| 契約開始日 / 更新予定日 | 月次定額のみ |
| 契約状態 | `契約中` / `解約`（+解約日） 月次定額のみ |

## 4. 金額・集計ロジック
- 案件 **月額合計(MRR)** = Σ 月次定額明細(契約中) の 単価×数量
- 案件 **単発合計** = Σ 単発明細 の 単価×数量
- 案件 **想定ACV(年換算)** = 単発合計 + 月額合計×12
- 案件 **加重ACV** = 想定ACV × 確度（※締結前パイプラインの評価に使用）
- 顧客ロールアップ：現MRR / ARR(=MRR×12) / 単発売上(受注済) / 契約中サービス数 / 商談パイプライン加重
- 全社KPI：現MRR・ARR・単発(受注済)・パイプライン加重、churn（解約MRR・解約率）

## 5. 画面
- **商談（nav）**：締結前カンバン（受注を目指す）。カード＝案件、金額はACV/MRR表示。
- **顧客（nav）＝ハブ**：一覧（MRR・契約中・パイプライン）＋詳細
  - 詳細：基本情報 / KPI(MRR・ARR・単発・契約中) / **契約一覧（=締結済み案件＋明細）** / 商談履歴（締結前） / 活動履歴 / 連絡先
- **案件詳細**：ヘッダに月額/単発/ACV、**明細エディタ**（行追加：サービス・課金タイプ・金額・数量・契約期間・契約状態）、活動ログ
- **ダッシュボード/月次/目標**：加重売上を **MRR** と **単発** に分離表示

## 6. 既存への波及・マイグレーション
- `Deal.expectedRevenue`(単一値)・`Deal.services`(テキスト) → **明細へ移行**。
  移行方針：既存案件は「単発1明細＝expectedRevenue、名称=services」で自動変換。
- 集計を使う全画面（カンバン/一覧/ダッシュボード/月次/目標/顧客）を**明細ベース**に改修。
- Supabase に `DealLineItem` 追加。破壊的変更は「①明細テーブル追加＋移行 → 新コード投入 → ②expectedRevenue/services 削除」の2段で無停止化。

## 7. 実装フェーズ
- **A. データ基盤**：DealLineItem＋契約/解約、enums(課金タイプ・フェーズ群)、追加マイグレーション、明細への移行seed
- **B. 案件UI**：明細エディタ、案件詳細の金額サマリ、カンバン/一覧をACV/MRR表示に
- **C. 顧客ハブ**：MRR/単発/契約中ロールアップ、契約一覧・商談履歴の分離表示
- **D. 集計**：ダッシュボード/月次/目標を MRR・単発分離に対応 ＋ 旧カラム削除マイグレーション＋デプロイ

## 8. 用語
- 締結前＝「商談」、締結後の Deal ＝顧客の「契約」。データ上は同一 Deal。
- 週次の「案件進捗管理(WeeklyProgress)」とは別概念（必要なら「制作進捗」に改称可）。
